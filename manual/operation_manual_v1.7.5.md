Fallout 76 自動解析ラボ 運用マニュアル v1.7.6
1. 目的
本ドキュメントは、Fallout 76 のゲーム内データを正確に抽出し、原子分解するための標準プロトコルを定義し、同時にラボの運用における安全性を確保するためのものである 。

2. ファイル取得プロトコル（命名規則）
データのトレーサビリティ確保と自動処理プロセスにおけるエラー防止のため、以下の命名規則を絶対的な正解とする。

厳格な命名規則: Fallout76_YYYY_MM_DD_NN.json

構成: [ゲーム名] + [西暦] + [月] + [日] + [連番2桁]

例: Fallout76_2026_02_02_01.json

運用ルール:

日付と連番をアンダースコア（_）で繋ぎ、枝番（NN）は 01 から開始する。

中間データおよびダウンロードファイルはこの形式以外を認めない。

3. 実行承認プロトコル（安全装置）

二段階承認: 捕捉した原本名と判定結果を表示し、博士の y/n 入力を待つ 。


強制終了: y 以外が入力された場合は即座にプロセスを破棄する 。

4. データ抽出・変換プロトコル（博士流エイリアス：原子分解版）
抽出されたデータは、プログラムに流し込む前に「博士流エイリアス」として原子分解および用語の統一を行う 。

原子分解の鉄則:

アンステイブル → 素早く・爆発 （※「高速」は永久追放） 。

スティングフロスト → 氷結・毒 。

冷酷無比 → 回復・弾力性 。

用語統一表:

鋭い視線 → Per増加 

サヴェージ・ストライク → アーマー貫通 

グループ再生 → 回復 

ウエストテック研究センター → ウエ研 

スーパーミュータント → スパミュ 

5. 技術提案・調査プロトコル (v1.0.1 統合)

5.1 打診義務: 新しい技術的解決策を提案する際は、手順やコードを提示する前に必ず「その件について調べましょうか？」と許可を求めること 。


5.2 最新状況の検証: 調査指示を受けた際は、「2025-2026年現在の最新のセキュリティ仕様」「リスク（ブロックの可能性等）」「コスト」を詳しく報告すること 。


5.3 実行承認: 博士から「実行」の承諾を得るまでは、コード作成や設定手順の提示に移らないこと 。

6. 管理・記録
新しいルールが決まったら、必ず「明文化しましたか？」「マニュアル化しましょうね？」と確認すること 。

Xの投稿作業については、第5項の調査プロトコルは適用されない。
